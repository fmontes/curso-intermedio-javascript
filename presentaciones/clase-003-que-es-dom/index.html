Entendamos el navegador

Empezando de arriba hacia abajo:

  • b>Navegador: representa el navegador como tal (Chrome, Firefox, etc). En JavaScript puedes accesarlo a traves del objeto `Navigator` y puedes obtener de el lenguaje, o la localizacion del dispositivo (previo permiso aprobado por el usuario), si tiene conexion a internet etc.
  • Ventana: Es el tab donde se carga la pagina web y se accesa a traves del objeto `Window`. Entre otras cosas te permite manipular el documento y escuchar eventos.
  • Documento: Es la pagina web como tal, es donde vive realmente el DOM. Te permite manipular la informacion del HTML y el CSS de la pagina y se puede accesar a traves del objeto `Document`.

¿Y el DOM?

Significa Document Object Model

Es una interfaz de programación para los documentos HTML y XML.

DOM

Nos da una representación del documento como un grupo de nodos y objetos estructurados.

DOM y JavaScript

  • El DOM no es un lenguaje de programación
  • Sin el DOM JavaScript no tendría como acceder y modificar una página Web
El DOM fue diseñado para ser independiente de cualquier lenguaje de programación y se hizo de tal manera que la presentación estructural del documento sea disponible desde un simple y consistente interfaz.

El arbol del DOM

Cuando un navegador web analiza un documento HTML, crea un árbol DOM y luego lo usa para mostrar el documento.

¿Objetos? ¿Que son?

Es un ente que tiene propiedades (caracteristicas) y metodos (comportamientos o acciones).

Acá nos estamos metiendo un poco en el paradigma de la programación orientada a objetos (que vamos a ver más adelante), pero necesitamos entender el concepto de objetos para entender mejor como funciona el DOM.

Todos los elementos del DOM son objetos

Vamos a modificarlos con ellos usando sus propiedades y sus metodos.

Tipos de Objetos del DOM

  • Document
  • Node
  • Element
  • NodeList

Document

Representa cualquier pagina web cargada en el browser

Representa cualquier pagina web cargada en el browser y sirve como punto de entrada al contenido de la pagina web.

Node

Todos los objetos dentro del Document es un nodo de algún tipo.

Element

Un elemento es un tipo específico de nodo

  • <a>: HTMLAnchorElement
  • <p>: HTMLParagraphElement
Un elemento es un tipo específico de nodo. Por ejemplo <a> es un element de tipo HTMLAnchorElement pero un <p> es un element de HTMLParagraphElement.

NodeList

Es una colección de nodos

es una coleccion de nodos. Es como una "especie de array", es decir puedes acceder a un nodo de la lista haciendo lista[4] como en un array, pero no tiene todos los metodos de un array
