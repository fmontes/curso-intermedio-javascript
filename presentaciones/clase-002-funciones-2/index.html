El resto de parámetros

Todos los argumentos no definidos en la función como un array


                        function saludar(saludo, ...nombres) {
                            return saludo + " " + nombres.join(", ") + "!";
                        }
                        
                        saludar("Hola Mundo", "Kanye", "West");
                        // "Hola Mundo, Kanye West!"

Representa todos los argumentos no definidos en la función como un array.

Cuando digo como un array significa que tenemos todos los metodos del array (map, filter, reduce) disponibles para usar.

Pero como el nombre lo dice "resto de los parámetros", esto signifca que el array va a contener los valores de los parámetros que no tengan nombre en la definicion de la función.

Y para crearlo crearlo o accesarlo hacemos ...nombreDeLaVariable en la lista de parametros de la función.

Todos los parametros


                        // args es el nombre del array
                        function sumarTodo(...args) { 
                            let sum = 0;
                          
                            // Iteramos sobre el array
                            args.forEach(function(arg) {
                              sum += arg; // sumamos los valores    
                            })
                          
                            return sum;
                          }
                          sumarTodo(1, 2, 3, 4, 5) // 15

Al no definir ningún parametro, solo el resto de ellos, obtenemos todos.

Scope

Es el ambito donde vive variable

Scope

Hay dos tipos

  • Global
  • Local

Scope Global

Variables definidas fuera de una función.

Scope Local

Variables definidas dentro de una función.

Scope

La función usa las variables del scope donde vive.


                        var nombre = "Freddy";
                        var apellido = "Montes";
                        
                        function imprimirDatos() {
                            var saludo = "Hello";
                            
                            console.log(nombre + ' ' +  apellido);
                            
                            function hello() {
                                console.log(saludo + ' ' + nombre + ' ' +  apellido);
                            }

                            hello();
                        }
                        
                        imprimirDatos();
                        // "Freddy Montes"
                        // "Hello Freddy Montes"
Las variables definidas fuera de la función pero dentro del scope global son accesibles desde la función.

Scope

Pero el scope no puede usar las variables de la función


                        var nombre = "Freddy";
                        var apellido = "Montes";

Scope de bloque

Variables con let y const tienen scope de bloque.

Con la introducción de let y const para declarar variables tenemos un nuevo scope, el de bloque. Podriamos decir que es una especie de "sub scope".

¿Que es un bloque?

Sentencias (lineas de código) entre {}

Bloques más comunes


                        if (some_condition) {
                            console.log('hacer algo');
                        }
                            
                        for (let arg of arguments) {
                            console.log('hacer algo');
                        }
                            
                        while (counter < 10) {
                            console.log('hacer algo');
                        }

let y const


                        let numero = 10;

                        if (numero > 9) {
                            let mensaje = 'El numero es mayor que 9';
                            console.log(mensaje);
                        }
                        
                        // ReferenceError: mensaje is not defined
                        console.log(mensaje);

Todas las variables declaradas con let y const tiene su scope en el bloque donde fueron declaradas.

Cambiamos por var


                        let numero = 10;

                        if (numero > 9) {
                          var mensaje = 'El numero es mayor que 9';
                          console.log(mensaje);
                        }
                        
                        // "El numero es mayor que 9"
                        console.log(mensaje);

Todas las variables declaradas con let y const tiene su scope en el bloque donde fueron declaradas.

Lo mismo en una función


                        function imprimirMensaje(numero) {
                            if (numero > 9) {
                                let mensaje = 'El numero es mayor que 9';
                                console.log(mensaje);
                                // El numero es mayor que 9
                            }
                            
                            console.log(mensaje);
                            // ReferenceError: mensaje is not defined
                        }
                          
                        imprimirMensaje(10);

Closure

Es cuando una fución interna tiene acceso a las variables de la función externa.

Es cuando una fución interna (o hija) puede recordar y tiene acceso a las variables de la fución externa (o padre).

Pero incluso despues que la función padre fué ejecutada.

¿Para que sirve closure?

Esta función suma 5 al valor que le pasemos


                        function agregarCinco(numero) {
                            return numero + 5
                        }
                        
                        agregarCinco(10) //15

¿Como hacemos una función que nos permita sumar cualquier numero?

Usando closure

Una función que retorna otra función


                        function agregarNumero(numero) {
                            function agregar(otroNumero) {
                                return numero + otroNumero;
                            }
                            return agregar;
                        }

Si vemos este codigo bien, tenemos una función que: Crea una función interna y luego la retorna.

Al retornar algo significa que podemos guardarlo en una variable.

¿y esto para que sirve?

Usando closure

Una función que retorna otra función


                        function agregarNumero(numero) {
                            function agregar(otroNumero) {
                                return numero + otroNumero;
                            }
                            return agregar;
                        }
                          
                        var agregar10 = agregarNumero(10);
                        var agregar5 = agregarNumero(5);
                          
                        agregar10(10) // 20;
                        agregar5(10) // 15;

Repasemos el concepto de closure:

Función que tiene acceso a las variables de su scope externo.

En este caso tenemos una función que dentro declara otra función (interna).

Esta función interna (`agregar`) tiene acceso no solo al Global Scope sino tambien al scope de su función externa `agregarNumero` y por ende tiene acceso a sus variables: el parámetro `numero`.

Entonces, cuando llamamos a la función `agregarNumero` con un `numero` y la guardamos en una variable (porque retorna algo, una función) lo que estamos guardando en dicha variable es la función `agregar` con el parámetro `numero` que pasamos en ese momento, en el momento de la llamada.

                    const RANDOM = 99;
